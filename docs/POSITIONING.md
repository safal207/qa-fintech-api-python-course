# Позиционирование курса

## Название

**QA в финтехе: API-тестирование и автоматизация на Python**

## Усиленная версия оффера

Научитесь тестировать платежи, балансы, комиссии, лимиты, idempotency и внешние интеграции. Соберите автотестовый проект на Python + Pytest + Allure + CI, который можно показать в резюме и использовать как основу для реальной работы в финтех-команде.

## Главная фраза

Этот курс не про “проверить 200 OK”.  
Он про то, как QA думает, когда ошибка может затронуть деньги пользователя.

## CTA для лендинга

- Основной: начать курс на Stepik.
- Пока курс не опубликован: открыть GitHub-проект и запустить sandbox локально.
- Вторичный: посмотреть программу и финальный проект.

## Целевая аудитория

1. Manual QA, который хочет перейти в API/automation.
2. QA, который хочет попасть в fintech.
3. Junior/Middle QA, которому нужен портфолио-проект.
4. Тестировщик, который хочет понимать платежи, балансы, лимиты, комиссии и транзакционные риски.

## Что продаёт курс

Не Python сам по себе.  
Не Pytest сам по себе.  
Не Postman сам по себе.

Продаёт курс вот это:

> Я умею тестировать API там, где ошибка стоит денег.

## Ключевые боли студента

- Не понимаю, как тестировать финтех-домен.
- Знаю теорию API, но не знаю реальные сценарии.
- Хочу перейти из ручного тестирования в автоматизацию.
- Хочу проект для GitHub и резюме.
- Боюсь собеседований по API, HTTP, SQL, fintech-сценариям.

## Главный результат

После курса студент должен уметь объяснить и показать:

- как тестировать перевод денег;
- как проверять баланс после операции;
- как проверять комиссии;
- как тестировать idempotency;
- как писать pytest API-тесты;
- как подключать Allure;
- как запускать тесты в CI;
- как описывать финтех-дефекты на языке бизнеса.
